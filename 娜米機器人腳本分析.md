## 一、整體架構概覽

此機器人服務「娜米」與「SheSay」兩個品牌，主要業務為**單身交友配對、戀愛諮詢**，並結合**贈品/優惠誘因**引導使用者預約諮詢或填寫問卷。

主要流程架構分為 5 大類型：
1. **訊息模組**（直接回覆訊息）
2. **問卷調查模組**（收集使用者資料）
3. **條件分流模組**（根據標籤或平台分流）
4. **互動遊戲模組**（輪盤抽獎、刮刮樂）
5. **子腳本**（獨立的子流程）

## 六、主要對話流程分析

### 主流程入口（開始使用）
```
開始使用
  └── 4. 開場分流（判斷平台）
       ├── FB → FB 娜米｜開場（新）
       └── LINE → LINE 相關流程
```

### FB 娜米 核心流程
```
FB 廣告入口（廣告新版/配對諮詢/活動專屬）
  └── 使用者回應
       ├── 有什麼活動 → FB 娜米｜有什麼活動
       ├── 有什麼對象 → FB 娜米｜有什麼對象
       │    └── 對象分流（單身未婚/離婚無子/離婚有子）
       │         └── FB 娜米｜配對基本資料（12題問卷）
       ├── 免費預約諮詢 → 預約分流
       │    ├── 已填寫 → 預約確認
       │    └── 未填寫 → 問卷填寫
       └── 免費索取戀愛數字報告
            └── 戀愛數字報告分流
                 └── FB 娜米｜戀愛數字報告問卷（男）
```

### LINE SheSay 核心流程
```
LINE 廣告入口
  └── 使用者回應
       ├── 有什麼活動 → LINE SheSay｜有什麼活動
       ├── 有什麼對象 → LINE SheSay｜有什麼對象
       ├── 免費預約諮詢 → LINE SheSay｜預約分流
       ├── 免費索取戀愛數字報告
       │    └── 戀愛數字報告分流
       │         └── LINE SheSay｜戀愛數字報告問卷
       └── 脫單漂流瓶 → 漂流瓶前的預約分流
            └── LINE SheSay｜脫單漂流瓶
```

### 廣告誘因類型（SheSay 女生廣告）
多種美食/娛樂廣告作為入口，引導女生用戶：
- 馬卡龍、夢幻蛋糕、蘋果塔（甜點系列）
- KTV、品酒、鮭魚料理（活動系列）

## 九、主要業務邏輯總結

1. **雙品牌運作**：娜米（偏向男性用戶）+ SheSay（偏向女性用戶）
2. **雙平台**：同時運行 FB Messenger 及 LINE
3. **核心轉化漏斗**：
   - 廣告吸引 → 開場互動 → 感情狀態分流 → 問卷收集 → 預約諮詢
4. **主要引流手段**：贈品（電子書、馬卡香水、戀愛數字報告）
5. **分流邏輯**：以「回答完畢」標籤判斷用戶是否已填寫問卷，避免重複引導
